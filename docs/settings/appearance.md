# 外观设置

路径：**阅读器标题栏 → 设置 → 外观设置**。

![外观设置页中的三种形态、主题颜色与结构参数](/screenshots/guide-07-appearance-settings.png)

## 主题模式

设置中心左下角提供三种模式：

- **明**：使用阅读器明亮配色；
- **暗**：使用阅读器暗色配色；
- **系统**：跟随宿主页面和系统当前主题。

主题模式立即生效。外观字段仍按明、暗主题分别保存适应色，避免同一颜色在另一主题中失去对比度。

## 配置范围

| 分组 | 典型字段 | 影响 |
| --- | --- | --- |
| 基础色 | 强调色、链接色 | 按钮、状态和正文链接 |
| 帖子斑马纹 | 颜色、圆角 | 主信息流帖子背景 |
| 宿主列表斑马纹 | 颜色、圆角 | LINUX DO 主题列表 |
| 结构颜色 | 总开关 | 关系线、引用线等自定义色是否启用 |
| 回复关系线 | 颜色、宽度、圆角 | 父子回复连接 |
| 引用线 | 颜色、宽度 | 引用块边界 |
| 分隔线 | 颜色、宽度 | 内容区块分割 |
| 楼层预览 | 颜色、高度 | 父回复和目标楼层预览 |

线宽允许约 0.5–4 px；楼层预览高度允许 180–720 px。

## 调整步骤

1. 先切到要核对的明/暗主题。
2. 选择浮窗、全屏或移动配置。
3. 调整强调色和链接色。
4. 调整帖子背景和圆角。
5. 开启结构颜色后，再设置关系线、引用线和分隔线。
6. 检查楼中楼、引用、普通楼层和已解决答案。
7. 点击“确认应用”。

## 共享规则

每个外观项目都可以单独共享。建议：

- 强调色和链接色跨形态共享；
- 帖子圆角按移动/桌面分开；
- 关系线宽度共享，楼层预览高度按形态保存；
- 明、暗主题分别核对，不要只在当前主题下判断。

## 可读性检查

- 正文与背景应有足够对比度。
- 链接不能只靠颜色区分，仍应保留阅读器的链接样式。
- 关系线过粗会抢占正文注意力；过细在高分屏上可能难以辨认。
- 斑马纹只用于帮助分段，不应覆盖回应、错误或已解决等语义色。

修改结果不理想时，先恢复当前项目默认值，不必重置全部设置。
