# 数据管理

路径：**阅读器标题栏 → 设置 → 数据管理**。

![数据管理页中的设置导出导入、恢复默认和缓存分类](/screenshots/guide-13-data-management.png)

## 设置配置

### 导出设置

导出文件包含规范化的阅读器配置，例如图片、灯箱、字体、布局、浮窗、外观、动效、性能和其他功能。

不包含：

- 登录凭据和 Cookie；
- 帖子正文；
- 浏览历史；
- 收藏、回应或原站账号数据；
- API 响应缓存。

### 导入设置

1. 先导出当前设置作为备份。
2. 点击“导入设置”并选择配置文件。
3. 核对格式和版本提示。
4. 确认导入。
5. 按提示刷新页面。

无效字段会被规范化或拒绝。导入不会把另一个浏览器的登录状态带到当前浏览器。

### 恢复全部默认

该操作覆盖当前阅读器设置，并通常需要刷新页面。它不等于清理缓存，也不会撤销原站收藏、回应或通知状态。

## 本地缓存

面板按六类展示数量、大小、保存位置和最长保留期。常见类别包括：

- 浏览历史；
- 主题与楼层；
- 用户卡；
- 消息分页；
- 图片与媒体；
- 其他可复用结构化数据。

实际名称和数量以当前版本面板为准。

## 选择性清理

1. 勾选需要清理的缓存类别。
2. 核对条目数、容量和保留期。
3. 点击清理。
4. 重新打开相关主题或面板，让阅读器按需重建数据。

清理主题与楼层缓存会增加下一次打开主题的请求；清理用户卡不会删除账号；清理消息缓存不会把通知标记为已读。

## 操作边界

| 操作 | 设置 | 本地业务记录 | 原站账号状态 |
| --- | --- | --- | --- |
| 导出设置 | 读取 | 不包含 | 不影响 |
| 导入设置 | 覆盖 | 不影响 | 不影响 |
| 恢复默认 | 覆盖 | 不影响 | 不影响 |
| 清理缓存 | 不影响 | 按选中类别删除 | 不影响 |
| 从收藏中心移除 | 不影响 | 同步更新 | 会改变 |

清理前先判断目标属于“配置”“缓存”还是“原站业务状态”，不要把三者混为一谈。
