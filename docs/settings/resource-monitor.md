# 资源监控

路径：**阅读器标题栏 → 设置 → 资源监控**。

![资源监控页中的基线、前后台实测和资源指标](/screenshots/guide-10-resource-monitor.png)

此页是只读诊断面板。它不会自动上传观测结果，也不会代替 Chrome DevTools 性能分析。

## 基线状态

打开面板后，阅读器先建立当前页面基线。基线用于区分：

- 监控开始前已存在的 DOM 和资源；
- 监控期间新出现的长任务、长帧脚本、DOM 变化和网络活动；
- 标签页进入后台后的真实时间空档；
- 浏览器没有提供的指标。

“浏览器未提供”不是 0；它表示当前环境无法直接测量。

## 最近 60 秒前后台实测

| 行 | 含义 |
| --- | --- |
| 阅读器 | 阅读器 Shadow DOM、元数据和相关资源 |
| 宿主 / 未标记 | LINUX DO 页面或无法可靠归因的活动 |
| 页面共享 | 标签页级别、不能只归到阅读器或宿主的指标 |

前台事件和后台事件分开统计。切到后台后，计时器可能被浏览器节流或冻结；监控会保留真实空档，不补造采样点。

## 重点指标

- DOM 数量与变化；
- 长任务；
- Long Animation Frame 脚本；
- 网络资源数量与传输；
- 页面可见性变化；
- 媒体、Object URL 和事件资源；
- 当前值、样本均值、观测峰值和首次变化。

## 诊断顺序

1. 先建立至少 10 秒基线。
2. 重现一次卡顿、滚动或媒体加载。
3. 看“首次变化”确认时间点。
4. 对比阅读器与宿主行。
5. 再到“请求数据”核对同一时间段是否有请求峰值或限流。

## 保留范围

趋势仅在当前页面内存中保留最近约 10 分钟。刷新或关闭页面后不会自动恢复；需要长期证据时，应在问题发生后立即截图或另行记录。
