# 楼中楼与上下文

LINUX DO 的回复既有正式楼层号，也可能属于某个父楼层的直属讨论。阅读器同时保留这两种关系。

## 父回复预览

点击楼层中的父回复提示或引用：

1. 阅读器优先从当前主题数据或缓存中定位源楼层。
2. 缺失时按需请求对应楼层。
3. 在当前位置展示预览，避免立即打断阅读。
4. 选择楼层号可跳到正式位置，返回时仍可找回原上下文。

![父楼层、直属回复、关系线和正式楼层同时保留](/screenshots/guide-18-thread-context.png)

<p class="image-caption">父楼层下方的缩进区域表示直属讨论；点击引用或父回复提示查看来源，点击楼层号跳到它在主信息流中的正式位置。</p>

## 楼中楼分页

直属回复默认每页显示 10 条，接口单次最多获取 20 条。阅读器只在父楼层接近视口后请求首批，后续页由翻页动作触发。

“设置 → 其他功能 → 回复展示”控制两个独立位置：

- **默认展开二级回复**：在父楼层下铺开直属回复。
- **展开二级回复对应楼层**：在主信息流正式楼层处完整显示该回复。

至少保留一种展开方式。两种都打开时，同一内容可能在父楼层下和正式楼层处各出现一次，这是关系表达，不是重复数据。

![展开后的楼中楼、关系线、回应和回复入口](/screenshots/guide-20-community-actions.png)

<p class="image-caption">展开二级回复后，先沿左侧关系线确认回复归属；每条直属回复仍可单独点赞、回应、Boost 或继续回复。</p>

## 引用内容

普通引用优先保留原文、作者和楼层链接。若来源楼层已删除或无权访问：

- 引用文本仍可显示；
- 已有图片节点可作为预览回退；
- 阅读器不会为 404/410 来源无限重试；
- 无法恢复的内容以不可用状态呈现。

## 已解决答案和特殊帖子

主题启用 Solved 等插件时，阅读器会显示已采纳答案卡片，并沿用正文的链接、SVG、媒体和 Onebox 样式。系统事件、指定、投票等非普通帖子也会以专门结构呈现；最终状态仍以原站数据为准。
