# 数据、配置与缓存

## 先区分三类数据

| 类型 | 示例 | 权威位置 | 清理结果 |
| --- | --- | --- | --- |
| 阅读器设置 | 图片比例、字体、布局、性能 | 当前浏览器 | 恢复显示和行为默认值 |
| 阅读器本地数据 | 历史、主题快照、用户卡、消息分页 | 当前浏览器 | 下次需要时重新请求；历史无法自动恢复 |
| 原站账号数据 | 帖子、消息、收藏、回应、已读状态 | LINUX DO | 只有对应业务操作才能改变 |

缓存清理不会撤销原站收藏或回应，也不会删除帖子和真实消息。

## 导出设置

“导出设置”生成 JSON，包含：

- 当前脚本版本；
- 图片和灯箱；
- 字体、布局、浮窗和外观；
- 跳转闪烁和等待动画；
- 性能、主题与其他开关；
- 规范化后的共享/独立形态配置。

不包含：

- 浏览历史；
- 帖子正文或 API 响应；
- 图片、头像和表情资源；
- Cookie、凭据或账号资料。

导入前建议先导出当前配置作为回退。

## 导入和恢复默认

- 只接受当前配置格式和版本结构；
- 导入会规范化数值和缺失字段；
- 无效文件不会部分写入；
- 成功导入或恢复全部默认后刷新页面。

“恢复全部默认”会覆盖当前阅读器设置。它不会清理所有缓存，也不会改动原站账号。

## 六类缓存

![数据管理中的缓存分类、保留期和选择性清理](/screenshots/guide-13-data-management.png)

| 类型 | 包含 | 不包含 |
| --- | --- | --- |
| 浏览历史 | 主题、最近楼层、查看时间 | 浏览器访问历史 |
| 主题与楼层 | 主题元数据、楼层正文、关系、分类标签图标 | 原站帖子 |
| 用户卡 | 简介、徽章、用户组、资料快照 | 头像二进制资源 |
| 消息 | 通知分页、已读摘要、跳转位置 | 原站真实消息 |
| API 响应 | 收藏、回应、点赞、表情注册表等 | 原站业务状态 |
| 头像与图片 | 头像、Boost 表情、实际查看过的原图 | 线上源文件 |

统计会区分业务对象和底层记录，并尽量去重；浏览器存储 API 的估算值不一定等于磁盘文件大小。

## 分层存储

阅读器按数据类型使用：

- 内存缓存；
- `localStorage`；
- Cache Storage；
- IndexedDB。

每层有最大条目、最大字节或保留期。新鲜数据直接读取，过期后联网校准；网络异常时，部分类型可以回退到仍在保留期内的旧数据。

## 安全清理顺序

1. 先导出设置。
2. 若只影响一个主题，优先清理当前主题缓存。
3. 若用户卡或消息显示旧值，只清理相应类型。
4. 只有配置本身异常时才恢复全部默认。
5. 清理后刷新页面并重新复现。

不要把“清空全部历史”用于一般加载故障，因为历史是本地唯一的阅读路径记录。
